# 战略资方合作管理系统

本仓库用于沉淀 Dowsure 战略资方合作管理系统的产品设计、项目档案、知识库和 MVP 演示。

系统第一阶段以 Standard Chartered Bank（SCB，渣打银行）作为样板间，先围绕真实合作场景跑通“战略资方管理 AI 工作台”的 MVP；但产品目标不是只服务 SCB，而是沉淀一套可复用的多资方管理底座。后续会尽快纳入 HSBC（汇丰）相关合作内容，并逐步扩展至 OCBC、PADB、Fusion Bank 等其他战略资方。

## 项目定位

战略资方合作管理系统不是单纯的 CRM，也不是传统项目管理工具，而是面向银行、资本方、金融机构及生态伙伴的战略合作 AI 工作台。

SCB 只是当前第一批业务样本，用来验证系统如何管理一个复杂资方的项目组合、客户推进、任务会议、知识库、风险卡点、资源投入和管理层汇报。系统设计需要天然支持多资方复用：同一套工作台框架，不同资方拥有独立的数据、知识库、项目线、指标口径和权限边界。

它希望解决以下问题：

- 多条合作线并行推进时，项目、客户、任务、会议、风险和指标分散在不同文档和表格中。
- 会议后行动项依赖人工跟进，容易遗漏、延期或责任不清。
- 管理层需要频繁查看合作健康度、目标进度、重大卡点和下一步动作。
- 各资方的合作知识、客户进展、产品方案和关键决策需要被持续沉淀、检索和复盘。
- 多家战略资方需要统一管理框架，但又必须按资方隔离数据、权限、指标和汇报视图。

## 资方样板与扩展

系统采用“通用底座 + 资方样板 + 持续抽象”的建设方式。

| 阶段 | 资方 | 定位 |
| --- | --- | --- |
| 第一阶段 | SCB / 渣打银行 | 作为首个样板间，验证复杂战略合作下的项目组合、客户转化、资源投入评分和汇报机制 |
| 第二阶段 | HSBC / 汇丰银行 | 尽快纳入相关合作内容，用于验证系统对不同资方项目结构、指标口径和知识库的适配能力 |
| 后续阶段 | OCBC、PADB、Fusion Bank 等 | 扩展为多资方战略合作管理底座，支持不同资方独立工作区与跨资方汇总视图 |

以 SCB 为例，当前 2026 年重点目标包括：

| 目标 | 说明 |
| --- | --- |
| eCAS 落地 | 完善 Dowalet 底层账户基础设施，支撑收还款、结算、后续放款和资金路径 |
| Project Dscale 规模化 | 推动大额跨境多币种信贷产品分层、标准化和数字化，目标形成超过 100M USD 资产池 |
| 多产品协同收入闭环 | 围绕 eCAS、Dscale、开户、Digital Onboarding、Affluent、MFP 等方向提升客户综合价值 |

## MVP 模块

第一版 MVP 聚焦 7 个核心模块：

| 模块 | 主要价值 |
| --- | --- |
| 资方总览仪表盘 | 查看单一资方或跨资方的目标、资产池、收入、开户、风险、里程碑和合作健康度 |
| 项目组合看板 | 按资方管理信贷、开户、账户体系、活动、资本协同等合作线 |
| 客户推进表 | 跟踪重点客户、开户客户、POC 客户、重点客户 LTV 和多产品转化路径 |
| 任务与会议 Follow-up | 将会议纪要拆解为待办事项，跟踪负责人、截止时间、状态和逾期提醒 |
| 知识库 / AI 问答 | 沉淀会议、文档、产品方案、客户进展、关键决策，并支持总结与汇报 |
| 项目管理档案库 | 分类归档背景信息、协议文档、会议纪要、产品资料和客户资料 |
| 市场活动及品牌共建 | 管理联合活动、品牌露出、邀约名单、物料库和活动复盘 |

## 样板项目线

SCB 作为当前样板资方，第一版项目组合看板建议包含：

- eCAS 账户体系
- Project Dscale
- 企业开户 / Digital Onboarding
- 企业主 / 高管个人开户
- MFP 长期合作

SCV 投资协同、Referral Agreement 等内容暂作为战略背景、重大里程碑或协议资料进入项目管理档案库，不作为日常项目组合看板中的执行项目线。

后续纳入汇丰后，系统应基于汇丰实际合作资料生成对应的项目线、客户推进路径、资源投入指标和汇报视图，而不是照搬 SCB 的项目结构。

## 资源投入评分

系统建议采用 1-5 分机制评估各战略资方对 Dowsure 的资源投入。SCB 当前作为首个评分样本，后续汇丰及其他资方可以复用同一套一级维度，并根据实际合作特点调整二级指标和权重：

| 维度 | 默认权重 | 管理含义 |
| --- | --- | --- |
| 产品资源 | 40% | 是否形成有竞争力、可落地、可规模化的产品能力 |
| 组织资源 | 35% | 是否有足够资方内部资源、owner、机制和时效保障 |
| 品牌资源 | 15% | 是否通过联合活动和对外背书形成市场影响力 |
| 资本资源 | 10% | 作为战略背景和重大里程碑观察，不作为日常高频管理重点 |

## 后续路线

- 接入飞书 OAuth、飞书 Wiki、群聊、任务、多维表格和日历会议纪要。
- 支持按资方工作区隔离知识库、客户表、任务、会议纪要和汇报视图。
- 尽快纳入汇丰相关合作内容，验证第二个资方工作区、项目线生成和指标适配。
- 将会议纪要自动拆解为任务，识别 owner、截止时间、依赖事项和风险信号。
- 建立管理层仪表盘，自动生成周报、月报和专项汇报。
- 从 SCB 样板间扩展为多资方战略合作管理底座，并支持跨资方资源投入、客户转化和合作健康度对比。
