# SCB 战略合作管理系统

本仓库用于沉淀 Dowsure 与 Standard Chartered Bank（SCB，渣打银行）战略合作管理系统的产品设计、项目档案、知识库和 MVP 演示。

项目第一阶段以 SCB 作为样板间，围绕真实合作场景搭建“战略资方管理 AI 工作台”的 MVP；后续可扩展为支持 HSBC、OCBC、PADB、Fusion Bank 等多资方的战略合作管理底座。

## 项目定位

SCB 战略合作管理系统不是单纯的 CRM，也不是传统项目管理工具，而是面向战略资方合作的 AI 工作台。

它希望解决以下问题：

- 多条合作线并行推进时，项目、客户、任务、会议、风险和指标分散在不同文档和表格中。
- 会议后行动项依赖人工跟进，容易遗漏、延期或责任不清。
- 管理层需要频繁查看合作健康度、目标进度、重大卡点和下一步动作。
- SCB 合作知识、客户进展、产品方案和关键决策需要被持续沉淀、检索和复盘。
- 未来多家战略资方需要统一框架，但又必须按资方隔离数据和权限。

## 2026 核心目标

SCB 合作的 2026 年重点目标包括：

| 目标 | 说明 |
| --- | --- |
| eCAS 落地 | 完善 Dowalet 底层账户基础设施，支撑收还款、结算、后续放款和资金路径 |
| Project Dscale 规模化 | 推动大额跨境多币种信贷产品分层、标准化和数字化，目标形成超过 100M USD 资产池 |
| 多产品协同收入闭环 | 围绕 eCAS、Dscale、开户、Digital Onboarding、Affluent、MFP 等方向提升客户综合价值 |

## MVP 模块

第一版 MVP 聚焦 7 个核心模块：

| 模块 | 主要价值 |
| --- | --- |
| SCB 总览仪表盘 | 查看目标、资产池、收入、开户、风险、里程碑和合作健康度 |
| 项目组合看板 | 管理 eCAS、Dscale、开户 / Digital Onboarding、MFP 等合作线 |
| 客户推进表 | 跟踪 Dscale 客户、开户客户、POC 客户、重点客户 LTV 和转化路径 |
| 任务与会议 Follow-up | 将会议纪要拆解为待办事项，跟踪负责人、截止时间、状态和逾期提醒 |
| 知识库 / AI 问答 | 沉淀会议、文档、产品方案、客户进展、关键决策，并支持总结与汇报 |
| 项目管理档案库 | 分类归档背景信息、协议文档、会议纪要、产品资料和客户资料 |
| 市场活动及品牌共建 | 管理联合活动、品牌露出、邀约名单、物料库和活动复盘 |

## 当前项目线

SCB 第一版项目组合看板建议包含：

- eCAS 账户体系
- Project Dscale

## 资源投入评分

系统建议采用 1-5 分机制评估 SCB 对 Dowsure 的战略资源投入：

| 维度 | 默认权重 | 管理含义 |
| --- | --- | --- |
| 产品资源 | 40% | 是否形成有竞争力、可落地、可规模化的产品能力 |
| 组织资源 | 35% | 是否有足够 SCB 内部资源、owner、机制和时效保障 |
| 品牌资源 | 15% | 是否通过联合活动和对外背书形成市场影响力 |
| 资本资源 | 10% | 作为战略背景和重大里程碑观察，不作为日常高频管理重点 |

## 后续路线

- 接入飞书 OAuth、飞书 Wiki、群聊、任务、多维表格和日历会议纪要。
- 支持按资方工作区隔离知识库、客户表、任务、会议纪要和汇报视图。
- 将会议纪要自动拆解为任务，识别 owner、截止时间、依赖事项和风险信号。
- 建立管理层仪表盘，自动生成周报、月报和专项汇报。
- 从 SCB 样板间扩展为多资方战略合作管理底座。
