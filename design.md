# 战略资方管理系统设计文档

## 1. 项目定位

战略资方管理系统用于管理 Dowsure 与银行、资本方、金融机构等战略资方之间的长期合作关系。系统第一阶段以 SCB（Standard Chartered Bank，渣打银行）作为样板间，先跑通真实合作场景，再抽象为可复制的多资方管理底座。

第一版系统不是单纯的 CRM，也不是单纯的项目管理工具，而是一个面向战略合作的 AI 工作台：

- 管理资方合作全生命周期，包括目标、项目、客户、任务、会议、风险、资源投入、财务结果和复盘。
- 打通客户推进、产品协同、资方资源、内部团队协作和管理层汇报。
- 将分散在表格、飞书、会议纪要、知识库和人工经验中的信息沉淀为可检索、可追踪、可复盘的数据资产。
- 以 SCB 为 MVP，未来扩展至 HSBC、Olea、其他银行及资本合作方。

## 2. 建设目标

### 2.1 业务目标

- 支撑 SCB 2026 年核心目标落地：eCAS 落地、Project Dscale 规模化、多产品协同收入闭环。
- 统一管理多条合作线，减少信息分散、推进靠人工盯办、会议事项遗漏和汇报耗时。
- 建立资方资源投入评分机制，量化资方在产品、组织、资本、品牌等方面的实际投入。
- 将客户从单产品转化推进为多产品经营，包括 Dscale、开户、Digital Onboarding、eCAS、MFP、财富管理相关机会。
- 为管理层提供健康度、目标进度、关键风险、重大卡点和下一步动作的一屏总览。

### 2.2 产品目标

- 第一阶段做成“SCB 战略合作管理 AI 工作台”。
- 第二阶段抽象出“多资方管理底座”，支持不同资方的数据隔离、指标配置、项目模板和权限配置。
- 第三阶段接入外部系统，包括 CRM、飞书、文档库、数据仓库、邮件、日历和资方侧协作数据。

### 2.3 AI 目标

- 自动整理会议纪要、项目文档、客户进度和知识库内容。
- 自动拆解任务、识别 owner、截止时间、依赖事项和风险信号。
- 支持面向资方合作的问答、总结、周报、月报和专项汇报生成。
- 支持物料生成，包括活动海报文案、产品 one-pager、客户沟通话术、内部操作指南和管理层汇报材料。

## 3. 用户角色与权限

| 角色 | 核心诉求 | 主要权限 |
| --- | --- | --- |
| 系统管理员 | 配置资方、用户、权限、字段、指标和集成 | 跨资方管理 |
| PMO / 战略合作负责人 | 日常推进项目、任务、会议和风险闭环 | 管理本资方全部项目与任务 |
| 高管 / 管理层 | 查看合作健康度、目标进度、重大风险和需介入事项 | 查看汇总视图和重点项目 |
| BD / 业务负责人 | 管理客户池、商机、产品转化和业务收入 | 管理客户、商机和业务指标 |
| 客户经理 / 销售 | 跟进具体客户、材料、开户、进件、沟通和待办 | 查看和编辑自己负责的客户与任务 |
| 风控 / 运营 | 跟踪在贷、逾期、审批、流程卡点和合规材料 | 查看风险、流程和客户材料 |
| 市场 / 品牌 | 管理活动、联合传播、物料生成和品牌资源投入 | 管理活动与物料 |

权限设计原则：

- 默认按资方隔离数据。
- 同一资方内按角色、项目、客户归属控制查看和编辑范围。
- 管理层默认查看汇总和重点事项，不暴露不必要的客户细节。
- AI 问答和材料生成必须遵守用户权限，只能使用当前用户有权访问的数据。

## 4. MVP 范围

第一版 MVP 建议聚焦 6 个核心模块。

| 优先级 | 模块 | 第一版目标 |
| --- | --- | --- |
| P0 | PMO 工作台 | 作为默认首页，聚合今日待办、逾期事项、项目卡点、客户提醒、会议 follow-up 和 AI 快捷入口 |
| P0 | 项目组合管理 | 管理 eCAS、Dscale、开户 / Digital Onboarding、MFP、GBA Summit、SCV 投资协同等合作线 |
| P0 | 客户推进表 | 跟踪 Dscale 客户、开户客户、POC 客户和重点客户 LTV |
| P0 | 任务与会议 Follow-up | 支持手动建任务、关联对象、负责人、截止时间、状态、逾期提醒 |
| P1 | 知识库 / AI 问答 | 沉淀会议、文档、产品方案、客户进展和关键决策，支持检索、总结和问答 |
| P1 | 管理层仪表盘 | 展示目标进度、资产池、收入、开户、风险、资源投入评分和合作健康度 |

物料生成中心作为 P1 / P2 能力保留入口，第一版可先支持文案草稿生成，不必完整实现设计排版流程。

## 6. 核心页面设计

### 6.1 默认首页：PMO 工作台

PMO 工作台优先回答以下问题：

- 今天必须推进什么？
- 哪些任务已经逾期或即将逾期？
- 哪些项目出现卡点？
- 哪些客户需要跟进？
- 哪些会议事项尚未闭环？
- 哪些材料需要生成或更新？

核心组件：

| 组件 | 内容 |
| --- | --- |
| 今日待办 | 当前用户负责或需协同的任务 |
| 逾期 / 风险事项 | 逾期任务、临近 deadline、需升级事项 |
| 项目卡点 | 每条项目线当前最大阻塞点、owner、下一步动作 |
| 客户推进提醒 | Dscale、开户、POC、重点客户的下一步动作 |
| 会议 Follow-up | 最近会议拆出的行动项及闭环状态 |
| 待生成材料 | 周报、活动文案、one-pager、客户话术等入口 |
| AI 对话悬窗 | 新建任务、问知识库、总结页面、拆解纪要、生成材料 |

### 6.2 资方总览页

资方总览页用于展示某一资方的整体合作状态。

建议内容：

- 资方名称、合作阶段、合作起点、当前战略定位。
- 年度目标及完成进度。
- 合作健康度：正常、关注、风险、严重风险。
- 资源投入评分：产品资源、组织资源、资本资源、品牌资源。
- 关键项目状态：eCAS、Dscale、开户、Digital Onboarding、MFP、活动、投资协同。
- 关键指标：资产池规模、授信、放款、收入、开户数、客户转化、逾期风险。
- 重大里程碑：已完成、进行中、即将到期、延期。
- 需管理层介入事项。

### 6.3 项目详情页

每条合作线作为一个项目管理对象。

项目详情页字段：

| 字段 | 说明 |
| --- | --- |
| 项目名称 | 如 eCAS、Project Dscale、企业开户 |
| 项目类型 | 产品合作、客户转化、活动、资本协同、技术对接 |
| 当前阶段 | 立项、POC、试点、规模化、复盘、暂停 |
| 项目 owner | Dowsure owner 与资方 owner |
| 成功标准 | 阶段验收标准和业务结果 |
| 关键里程碑 | 时间节点、状态、完成证据 |
| 当前卡点 | 卡点描述、责任方、预计解决时间 |
| 关联客户 | 与项目相关的客户、商机和案例 |
| 关联任务 | 所有未完成和已完成任务 |
| 关联文档 | 方案、纪要、合同、物料、邮件摘要 |
| 风险等级 | 低、中、高、严重 |

### 6.4 客户推进页

客户推进页用于跟踪客户在资方合作产品中的转化路径。

第一版关注路径：

```text
Dscale 重点客户 / 开户 POC 客户 / GBA Summit 高质量客户
  -> 企业开户或开户意向
  -> Dscale 授信需求
  -> eCAS 账户承接
  -> 企业主 / 高管个人开户
  -> 多产品转化与收入贡献
```

核心字段：

| 字段 | 说明 |
| --- | --- |
| 客户名称 | 企业客户名称 |
| 客户类型 | Dscale、开户、POC、活动客户、重点 LTV 客户 |
| 年 GMV / 销售额 | 客户规模判断 |
| 准入状态 | 是否满足资方准入标准 |
| 当前阶段 | 线索、意向、材料准备、进件、审批、开户、授信、放款、复盘 |
| 产品覆盖 | eCAS、Dscale、企业开户、个人开户、MFP、Affluent |
| 下一步动作 | 当前最重要推进事项 |
| Dowsure owner | 内部负责人 |
| 资方 owner | 银行 RM 或相关负责人 |
| 预计收入 | 利差、服务费、referral fee 等 |
| 风险 / 卡点 | 材料、审批、定价、合规、客户意愿等 |

### 6.5 任务与会议页

第一版任务系统先实现可靠的手动闭环，后续再逐步智能化。

任务字段：

| 字段 | 说明 |
| --- | --- |
| 任务标题 | 简短描述 |
| 任务来源 | 手动、会议、文档、里程碑、客户下一步、飞书、系统提醒、AI 建议 |
| 负责人 | Dowsure owner |
| 协作人 | 内部或资方协作方 |
| 截止时间 | 用于提醒和逾期判断 |
| 状态 | 待处理、进行中、等待外部、已完成、已取消 |
| 优先级 | P0、P1、P2、P3 |
| 关联对象 | 资方、项目、客户、会议、文档、风险、材料 |
| 完成证据 | 链接、附件、备注、结果说明 |

任务来源演进：

| 阶段 | 能力 |
| --- | --- |
| MVP 1.0 | 手动添加任务、负责人、截止时间、状态、关联对象 |
| MVP 1.1 | 会议纪要和项目文档自动拆解任务 |
| MVP 1.2 | 项目里程碑倒推、客户下一步动作转任务 |
| MVP 1.3 | 飞书任务和重点群消息同步 |
| MVP 1.4 | 系统自动风险提醒 |
| MVP 1.5 | 智能建议 owner、截止时间、升级路径 |

### 6.6 AI 对话悬窗

对话悬窗在所有页面右侧常驻，默认收起，点击后展开。

第一版快捷按钮：

| 按钮 | 功能 |
| --- | --- |
| 新建任务 | 快速创建任务并关联当前页面对象 |
| 上传纪要 | 粘贴或上传会议纪要 |
| 生成材料 | 生成活动文案、one-pager、SOP、周报 |
| 问知识库 | 基于权限检索当前资方知识库 |
| 写周报 | 汇总项目、客户、任务、风险生成周报草稿 |

自然语言输入示例：

- “帮我给 eCAS 技术联调创建一个下周五前完成的任务，owner 是 Kevin。”
- “总结一下 Project Dscale 目前最大的三个卡点。”
- “把这段会议纪要拆成待办事项。”
- “基于 GBA Summit 信息生成一版客户邀约文案。”
- “列出过去 7 天没有更新的 Dscale 客户。”

## 7. 指标体系

### 7.1 SCB 样板核心指标

| 看板 | 核心问题 | 关键指标 |
| --- | --- | --- |
| Dscale 资产池 | 资产规模是否按目标增长 | 累计授信、累计放款、在贷余额、资产池规模、客户数 |
| 收入贡献 | Dowsure 从合作中获得了什么收入 | 放款利差 / 收入、referral fee、预计收入、已确认收入 |
| 开户转化 | 客户是否进入 SCB 账户体系 | 企业开户数、个人开户数、开户成功率、开户阶段分布 |
| 战略资源 | 资方是否投入足够资源 | 产品资源、组织资源、资本资源、品牌资源评分 |
| 重点客户 LTV | 客户是否形成多产品转化 | 产品覆盖数、授信 / 放款、开户状态、后续机会、综合价值 |

### 7.2 资源投入评分

默认采用日常经营型权重：

| 维度 | 权重 | 含义 |
| --- | --- | --- |
| 产品资源 | 40% | 是否形成有竞争力、可落地、可规模化的产品能力 |
| 组织资源 | 35% | 是否有 owner、机制、审批效率和升级路径保障推进 |
| 品牌资源 | 15% | 是否通过联合活动和对外背书形成影响力 |
| 资本资源 | 10% | 是否形成投资、投后协同和集团级背书 |

评分采用 1-5 分：

| 分数 | 含义 |
| --- | --- |
| 1 | 仅口头表达支持，暂无实际投入 |
| 2 | 有初步讨论或个别支持动作，尚未形成机制 |
| 3 | 有明确方案、owner 或阶段性动作，能支持单点项目推进 |
| 4 | 形成稳定机制，能持续支持多个客户或项目 |
| 5 | 成为战略级资源配置，具备可复制、可规模化、可对外背书能力 |

## 8. 数据模型

### 8.1 核心实体

```text
Funder 资方
  ├─ FundingRelationship 合作关系
  ├─ Project 项目
  ├─ Customer 客户
  ├─ Opportunity 商机 / 产品机会
  ├─ Task 任务
  ├─ Meeting 会议
  ├─ Document 文档
  ├─ Risk 风险 / 卡点
  ├─ Milestone 里程碑
  ├─ Metric 指标
  ├─ ResourceScore 资源投入评分
  └─ Material 物料
```

### 8.2 关键关系

- 一个资方可以有多个合作项目。
- 一个项目可以关联多个客户、任务、会议、文档、风险和里程碑。
- 一个客户可以同时关联多个产品机会。
- 一个任务必须至少关联一个对象，如项目、客户、会议、文档或风险。
- 一个会议可以生成多个任务和多个决策记录。
- 一个风险可以关联任务、项目、客户和升级记录。
- 一个指标可以归属于资方、项目、客户或产品线。

### 8.3 建议状态枚举

项目阶段：

```text
idea / planning / poc / pilot / scaling / live / paused / closed
```

任务状态：

```text
todo / in_progress / waiting_external / done / canceled
```

客户推进阶段：

```text
lead / qualified / intent / document_preparation / submitted / under_review / approved / rejected / onboarded / disbursed / retained
```

风险等级：

```text
low / medium / high / critical
```

资方合作健康度：

```text
healthy / watch / at_risk / critical
```

## 9. AI 能力设计

### 9.1 知识库问答

输入来源：

- 项目文档、会议纪要、客户进展、产品方案、历史汇报、活动材料、邮件摘要、飞书文档。

输出能力：

- 回答某个项目、客户、指标或风险的现状。
- 总结某条合作线的历史沿革。
- 找出关键决策、待确认问题和下一步动作。
- 生成管理层可读的简报。

### 9.2 会议纪要拆解

识别内容：

- 行动项、负责人、截止时间、依赖方、风险、决策、待确认问题。

输出内容：

- 待创建任务列表。
- 决策记录。
- 风险事项。
- 项目或客户状态更新建议。

第一版建议保留人工确认：AI 拆解后由用户选择“创建任务 / 忽略 / 修改”。

### 9.3 自动汇报

支持汇报类型：

- 周报
- 月报
- 项目专项汇报
- 管理层摘要
- 风险升级说明
- 客户转化进展
- 活动复盘

汇报结构建议：

```text
本期进展
关键指标
已完成事项
未闭环事项
风险与卡点
需要管理层决策 / 支持
下周计划
```

### 9.4 物料生成

第一版优先生成文本材料：

- 活动邀约文案
- 产品 one-pager 文案
- 客户沟通话术
- 内部 SOP 草稿
- 管理层汇报初稿

后续再接入设计模板、品牌规范和图片生成能力。

| 飞书文档 | 项目文档、会议纪要、知识库同步 |

| 飞书群 | 识别承诺、任务、卡点和待跟进事项 |
| 数据仓库 | 授信、放款、收入、开户、逾期等业务指标 |
| 日历 | 会议同步、会议 follow-up |
| 邮件 | 资方沟通摘要、附件归档 |

## 11. 非功能设计

### 11.1 数据隔离

- 多资方数据默认隔离。
- 用户只有在被授权后才能访问某资方数据。
- AI 检索必须执行同样的权限过滤。
- 管理层跨资方视图需要单独授权。

### 11.2 审计与追踪

需要记录：

- 任务创建、修改、完成、取消。
- 项目状态变更。
- 客户阶段变更。
- 指标数据更新。
- AI 生成内容的来源、操作者和确认状态。
- 关键文档上传、删除和版本变化。

### 11.3 可配置能力

为支持多资方复制，以下内容应配置化：

- 资方类型。
- 项目模板。
- 客户阶段。
- 指标口径。
- 资源评分维度和权重。
- 任务类型。
- 风险等级。
- 汇报模板。
- 权限角色。

## 12. 技术架构建议

### 12.1 应用层

- Web 管理后台，优先服务 PMO、业务负责人和管理层。
- 响应式布局，移动端第一阶段只保证可查看和简单处理任务。
- 默认首页为 PMO 工作台。

### 12.2 服务层

建议拆分为：

- 资方与项目服务
- 客户与商机服务
- 任务与会议服务
- 指标与仪表盘服务
- 知识库与文档服务
- AI 编排服务
- 权限与审计服务

### 12.3 AI / RAG 层

建议能力：

- 文档解析与结构化抽取。
- 向量检索与关键词检索混合召回。
- 权限过滤后再检索或检索结果二次过滤。
- 引用来源展示，避免无依据回答。
- AI 输出需标记为“草稿 / 待确认 / 已确认”。

### 12.4 数据层

建议组合：

- 关系型数据库：存储资方、项目、客户、任务、指标、权限等结构化数据。
- 对象存储：存储附件、会议纪要、汇报材料、物料文件。
- 搜索索引：支持全文检索。
- 向量库：支持语义检索。

## 13. MVP 迭代路线

### MVP 1.0：手动任务闭环与 SCB 工作台

目标：让 PMO 可以日常使用。

范围：

- 资方总览
- PMO 工作台
- 项目组合管理
- 客户推进表
- 手动任务创建与状态跟踪
- 基础风险与卡点记录
- 管理层仪表盘静态 / 半自动数据录入
- AI 对话悬窗入口保留，先支持简单问答或跳转

### MVP 1.1：知识库与会议自动化

目标：减少会议后整理成本。

范围：

- 文档上传和知识库检索
- 会议纪要拆解任务
- 决策与待确认事项提取
- 项目和客户状态更新建议
- 周报草稿生成

### MVP 1.2：业务流程自动化

目标：让项目、客户和任务之间自动联动。

范围：

- 里程碑倒推任务
- 客户下一步动作转任务
- 长期未更新提醒
- 临近 deadline 提醒
- 资源投入评分证据沉淀

### MVP 1.3：外部系统同步

目标：减少人工录入。

范围：

- CRM 客户同步
- 飞书文档同步
- 飞书任务同步
- 重点群消息摘要与任务识别
- 数据仓库指标同步

### MVP 1.4：智能预警与建议

目标：从记录系统升级为主动管理系统。

范围：

- 项目延期风险识别
- 客户转化停滞识别
- 资方响应超时识别
- owner 建议
- 升级路径建议
- 管理层介入建议

## 14. 第一版验收标准

第一版可用的最低标准：

- PMO 能在一个页面看到今日待办、逾期事项、项目卡点和客户推进提醒。
- 每条 SCB 合作线都有 owner、阶段、里程碑、卡点、下一步动作和风险等级。
- 每个重点客户都有当前阶段、产品机会、下一步动作和负责人。
- 会议后行动项可以被记录为任务，并能追踪负责人、截止时间和完成状态。
- 管理层能看到 SCB 合作目标进度、核心指标、风险事项和需介入事项。
- 知识库能沉淀文档，并支持按项目、客户、主题检索。
- 系统保留多资方扩展能力，SCB 数据结构不应写死为单一资方专用。
